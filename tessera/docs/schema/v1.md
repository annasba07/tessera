# Per-session narrative schema — v1

The unit of analysis is one session. For each qualifying session we produce
one JSON artifact combining deterministic event-stream metadata with an
LLM-extracted narrative. These artifacts are the input to the cross-session
synthesis pass and the long-term dataset for deriving empirical detectors
in v2.

## Field reference

### Deterministic block — extracted from `events.jsonl`, never LLM

#### Identity
| Field | Type | Notes |
|---|---|---|
| `session_id` | str | `<agent>:<uuid>` |
| `agent` | str | `claude` / `codex` / `gemini` |
| `project_path` | str | full cwd from events |
| `project_label` | str | last 2 path segments, dedup-friendly across worktrees |
| `git_branch_last` | str | last branch seen in events |
| `models_used` | list[str] | sorted unique model IDs |
| `primary_model` | str | model with most events |

#### Time
| Field | Type | Notes |
|---|---|---|
| `started_at` / `ended_at` | iso str | first / last event timestamps |
| `wall_clock_minutes` | float | raw first→last (often misleading; see `bursts`) |
| `active_minutes` | float | sum of inter-event gaps capped at 5min — closest to "real" duration |
| `bursts` | int | active windows separated by >5min idle |
| `primary_burst_minutes` | float | longest single burst |
| `longest_idle_minutes` | float | biggest gap inside the session |
| `events_per_active_minute` | float | pace |
| `time_of_day_buckets` | dict | counts by morning/afternoon/evening/night |
| `weekday` | str | day of week of `started_at` |

#### Volume
| Field | Type | Notes |
|---|---|---|
| `event_count` | int | |
| `tool_call_count` | int | |
| `user_turn_count` | int | user messages |
| `subagent_count` | int | distinct subagents spawned |
| `subagent_event_count` | int | events that happened inside subagents |
| `subagent_types_spawned` | dict | Counter of subagent_type if available |

#### Tool / scope
| Field | Type | Notes |
|---|---|---|
| `tool_call_distribution` | dict | top 10 tool names → count |
| `tool_category_distribution` | dict | browser/shell/read/write/delegation/web/other → count |
| `tool_error_rate` | float | errors / tool_calls |
| `tool_error_classes` | dict | Counter of error_class |
| `most_repeated_tool_call` | dict | `{tool, input_hash, count}` for the most-repeated `(tool, input)` pair |
| `unique_files_touched` | int | distinct file paths from Read/Edit/Write inputs |
| `top_files_touched` | list | top 10 file paths with frequency |
| `directory_concentration` | float | fraction of file ops in single dominant directory |

#### Verification proxies
| Field | Type | Notes |
|---|---|---|
| `tests_invoked` | int | bash matching `pytest|jest|vitest|cargo test|go test|npm test|ruff|mypy|tsc` |
| `git_status_invoked` | int | |
| `git_diff_invoked` | int | |
| `pr_referenced` | bool | tool input contains `gh pr` or `pull_request` |

#### Conversation shape
| Field | Type | Notes |
|---|---|---|
| `assistant_chars` | int | total assistant message chars |
| `user_chars` | int | total user message chars |
| `reasoning_chars` | int | total `<thinking>` chars |
| `assistant_to_user_text_ratio` | float | |

#### Behavioral signals
| Field | Type | Notes |
|---|---|---|
| `user_friction_signals.explicit_corrections` | int | user msgs matching `{"no", "wrong", "stop", "actually", "that's not"}` |
| `user_friction_signals.rephrased_retries` | int | user msgs with >50% word overlap to a prior user msg within 10 turns |

#### Tail / outcome proxies
| Field | Type | Notes |
|---|---|---|
| `last_event_type` | str | `tool_call` / `tool_result` / `message` / `reasoning` |
| `final_tool_status` | str | success / error / N/A — status of last tool result |
| `error_concentration_tail` | float | errors_in_last_10pct / errors_in_first_10pct |
| `events_since_last_user_message` | int | autopilot indicator |

### LLM-extracted block — Sonnet 4.6, structured JSON

#### Session-level
| Field | Type | Constraint |
|---|---|---|
| `goal` | str | ≤200 chars; one-line overarching goal of the session |
| `topics` | list[str] | 3-7 free-form short tags (the only open vocab) |
| `external_systems_touched` | list[str] | controlled vocab |
| `verification_completeness` | enum | `none / claimed_only / lightly_verified / thoroughly_verified` |
| `prompt_quality_signal` | enum | `underspecified / well_formed / overspecified` |

#### Tasks (≤5 per session)
| Field | Type | Constraint |
|---|---|---|
| `task_id` | str | `t1`, `t2`, ... |
| `intent` | str | ≤200 chars, specific |
| `event_range` | [int, int] | `[start_seq, end_seq]` valid indices |
| `task_type` | enum | controlled vocab |
| `task_difficulty` | obj | dimensional, see below |
| `outcome` | enum | controlled vocab |
| `outcome_evidence_event` | int | valid event index |
| `time_to_first_verified_progress` | int | event index, or `-1` |

##### `task_difficulty` (dimensional)
| Dimension | Values |
|---|---|
| `scope` | `single_file / few_files / cross_module / system_wide` |
| `specification` | `clear / partial / exploratory` |
| `external_dependencies` | `none / one / multiple` |
| `verification_ease` | `automated / partial / requires_human` |
| `overall` | `trivial / moderate / hard / very_hard` |

The LLM scores the 4 dimensions independently, then assigns `overall` consistent with them. Anchored examples in the prompt:

- **trivial**: 1 file, clear spec, no external deps, automated verification. *e.g. rename function, fix typo, format file.*
- **moderate**: 1-3 files, clear spec, may need context reading, ~10-30min for a competent dev. *e.g. add CLI flag, update test for new behavior, fix known bug.*
- **hard**: ≥4 files or cross-module, ambiguous spec, may involve external systems, 1-3 hours. *e.g. implement feature backend+frontend, debug intermittent failure, migrate schema.*
- **very_hard**: large scope, exploratory spec, deep external dependencies, often spans sessions. *e.g. new module from scratch, performance regression of unknown cause, architectural refactor.*

#### Friction (≤6 per session)
| Field | Type | Constraint |
|---|---|---|
| `event_range` | [int, int] | valid bounds |
| `task_id` | str | references a `tasks[].task_id` |
| `type` | enum | controlled vocab |
| `tool_category` | enum | controlled vocab |
| `description` | str | ≤200 chars |
| `key_quote` | str | verbatim from one event in `event_range` |
| `cost_events` | int | `= event_range[1] - event_range[0] + 1` |
| `cost_active_minutes` | float | derived from event timestamps |

`waste_signature` (session-level): the dominant friction `type`, or `none` if the session was clean.

**Extraction approach**: pure LLM-extracted in v1 — no deterministic scaffolding. Cast a wide net and let the model find shapes we haven't anticipated. v2 plan: derive empirical detectors from the v1 dataset once we have ~500+ extracted sessions.

#### Decisions (≤5 per session)
| Field | Type | Constraint |
|---|---|---|
| `event_index` | int | valid index |
| `decision` | str | what was chosen |
| `retrospective` | str | hindsight evaluation, with evidence |

#### Dead ends (≤3 per session)
| Field | Type | Constraint |
|---|---|---|
| `event_range` | [int, int] | valid bounds |
| `approach` | str | what was tried |
| `abandoned_at` | int | valid event index |
| `lesson` | str | what was learned |

#### Recurring environmental issues (≤3 per session)
| Field | Type | Constraint |
|---|---|---|
| `description` | str | name the underlying cause + the one-time fix |
| `occurrences` | list[int] | ≥2 event indices where it bit |

Distinct from `friction_moments`: a friction moment is a *time window of waste*, an environmental issue is a *root cause that bit ≥2 times with a clear one-time fix* (stale lock, expired auth, missing dep, wrong permission scope, MCP server crashed). Two browser_spiral friction_moments with the same Chrome SingletonLock cause = 2 friction moments + 1 environmental issue.

LLM has access to `tool_error_classes` in the deterministic header but it's surfaced as facts, not as a "look here" hint.

#### User caught model errors
| Field | Type | Constraint |
|---|---|---|
| `count` | int | total times user explicitly corrected a model claim |
| `examples` | list (≤3) | `{event_index, what_user_caught}` |

#### Lessons
| Field | Type | Constraint |
|---|---|---|
| `counterfactual` | str | 30-300 chars, must reference specific events |
| `lesson_for_agent` | str | ≤200 chars, what Claude should do on similar future tasks |
| `lesson_for_user` | str | ≤200 chars, concrete setup change only (not "be more careful") |
| `notable` | str | optional, ≤250 chars, surprising observation that doesn't fit elsewhere |

### Provenance
| Field | Type | Notes |
|---|---|---|
| `narrative_extracted_at` | iso str | |
| `narrative_model` | str | model ID |
| `schema_version` | int | bump invalidates cache |
| `events_content_hash` | str | sha256 for cache key |
| `narrative_quality` | enum | `low / medium / high` based on validation drop rate |

## Validation rules

Applied post-LLM, fail closed at the field level (drop the offending field, keep the rest of the session).

1. Every `event_range: [a, b]` satisfies `0 ≤ a ≤ b < event_count`. Drop offending friction_moment / key_decision / dead_end / task.
2. `outcome_evidence_event`, `time_to_first_verified_progress`, `key_decisions[].event_index`, `recurring_environmental_issues[].occurrences[]`, `user_caught_model_errors.examples[].event_index` must be valid indices. If invalid, blank that field (downgrade affected `outcome` to `unclear`).
3. `key_quote` substring-matches (normalized whitespace) one event in its `event_range`. If not, blank it.
4. All enum fields must be in their controlled vocab. Otherwise replace: `other` for `task_type` / `tool_category`, `unclear` for `outcome`, `none` for waste signatures.
5. `friction_moments[].cost_events == event_range[1] - event_range[0] + 1`. Otherwise recompute deterministically.
6. `friction_moments[].task_id` must reference an actual `tasks[].task_id`. Otherwise blank.
7. `tasks[]` event_ranges should be non-overlapping and cover all events. If not, log warning but don't drop.
8. `counterfactual` must be 30-300 chars. Otherwise blank.
9. `goal` ≤200 chars; `lesson_for_agent` and `lesson_for_user` ≤200 chars each.
10. `topics` must have 3-7 items, each ≤30 chars. Otherwise truncate.
11. `task_difficulty.overall` must be roughly consistent with its 4 dimensions (sanity check, not strict).
12. List length caps: `tasks ≤ 5`, `friction_moments ≤ 6`, `key_decisions ≤ 5`, `dead_ends ≤ 3`, `recurring_environmental_issues ≤ 3`, `user_caught_model_errors.examples ≤ 3`. Truncate excess.
13. `narrative_quality` = `low` if >50% of LLM fields invalidated, `medium` if 10-50%, `high` if <10%.

## Cache

Key: `sha256(events_content) + schema_version + narrative_model_id`

Path: `~/.cache/tessera/narratives/<session_id>.json`

Re-running after a normalizer fix only re-extracts changed sessions. Bumping `schema_version` invalidates all caches.

## Default filters for historical run (configurable)

- Skip sessions with `event_count < 20` (raise from default 4 — short sessions have no narrative)
- Skip sessions whose first user prompt is a slash-command artifact (`/tessera`, `/plugin`, `/compact`, etc.)
- Skip sessions older than `--max-age-days N` (no default cap)
- Dedupe by `(agent, session_id)` keeping the largest event count

## Cost & scale (full historical estimate)

| Metric | Estimate |
|---|---|
| Historical sessions across all agents | ~1500-2500 |
| Per-session input tokens | ~30-50K compressed event stream + ~3K metadata |
| Per-session output tokens | ~3K |

| Full historical cost | $75-$300 one-time |

| Resumeable | Yes (cache survives interruption) |

## Explicitly deferred to v2

- Continuation linkage between sessions (deterministic match in synthesis stage)
- Git ground truth (commits made, reverts, files actually shipped)
- File-level cross-session patterns (e.g. "`prompt_builder.py` keeps coming up")
- `agent_overconfidence_moments` / `agent_proactivity_misses` (hallucination risk)
- `agent_competence_this_session` (too noisy per-session)
- Empirical friction detectors derived from v1 data
- Synthesis-stage clustering and goal-arc detection across sessions
